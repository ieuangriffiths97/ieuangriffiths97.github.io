Dissertation Implementation



Project Overview

For my 3rd year dissertation I developed an iOS application in xCode, using the Swift 4 language. The application allowed users to log their weightlifting one-rep-max records over a period of time, and based on their level of progress reccomend the user tips to help them progress.

The software development methodology was the Waterfall model, as it was a University assignment and deadlines needed to be met, but in the real world I would of taken an Agile approach. I followed the following stages from the Software Development Life Cycle:

  • Requirements Analysis- Which was mostly done via a Literature Review
  • Design- Where I produced UI mock-ups and outlined the objects, classes and other programming needed to be done
  • Implementation- Where I developed the app itself
  • Testing & Evaluation- Which was done via: Asability tests for testing the user experience the app gave, Validation testing via a thorough test plan to ensure the app worked and Verification testing via testing the app against the function requirements to ensure the app was made right & did was it was supposed to
  • Put existing knowledge of SQL statements into practice, using them to connect to a database, store user submitted data and retrieve existing data
  • Gained experience in using source control within a team project. We used gitlab for source control.

Although efforts was made to ensure the app was easy to use and was in line with industry standards (such as Neilsen's heuristics and Krug's usability theories) the aesthetics of the app were not a primary concern and therefore minimal time and effort went into styling the app.

Additionally, the app had a feature that allowed users to create an account and have records unique to them, and share their progress to Facebook
